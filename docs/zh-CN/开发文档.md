# 开发文档

## 架构

项目是 Qt Widgets 桌面应用，采用源码优先的 CMake 结构。

主要模块：

- `premium-content-radar`：桌面主程序。
- `radar_core`：核心静态库，包含记录结构、评分、存储、设置、导出、插件管理和本地桥辅助逻辑。
- `WeChatProviderPlugin`：通过 `QPluginLoader` 加载的微信内容供应商插件。
- `radar_tests`：Qt Test 测试套件，覆盖存储、导出、评分和桥解析。

## 数据流

1. 应用启动后从用户配置 JSON 读取运行设置。
2. 插件管理器扫描配置中的插件目录。
3. 内容供应商启动后通过 `drainRecords()` 暴露文章记录。
4. 主程序按 URL 去重写入 SQLite。
5. 表格通过 `PremiumContentFilterProxyModel` 进行筛选和评分。
6. 导出模块从数据库查询结果生成 CSV 或 JSON。

## 运行配置

`AppSettingsController` 负责默认值和持久化，保存：

- SQLite 数据库路径。
- 插件目录。
- 本地桥端口。
- ADB 自动化开关。
- 是否启动时加载示例数据。

主程序加载插件前，会设置：

- `PREMIUM_RADAR_BRIDGE_PORT`
- `PREMIUM_RADAR_ENABLE_ADB`

微信插件读取这两个环境变量决定桥端口和是否启动 ADB 线程。

## 线程模型

- UI 在 Qt 主线程运行。
- `ProxyTrafficBridge` 使用 localhost `QTcpServer` 和互斥保护的双缓冲队列。
- `AdbAutomationEngine` 是独立 `QThread`，默认关闭。
- SQLite 写入由主程序线程执行。

## 本地桥协议

接受端点：

- `/mp/getappmsgext`
- `/mp/appmsg_comment`

接受字段：

- `read_num`
- `like_num`
- `old_like_num`
- `comment_num`
- `comment_count`
- 嵌套 `appmsgstat`
- 嵌套 `comment_info`

未知端点返回空记录，不入库。

- 不提交运行数据库、日志和打包产物。
- 英文文档不混入中文。
- 中文文档集中放在 `docs/zh-CN/`。
- 不把账号凭证、Cookie、私有流量写入源码或文档。
