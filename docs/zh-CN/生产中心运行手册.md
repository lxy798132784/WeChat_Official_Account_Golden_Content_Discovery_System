# 生产中心运行手册

本文说明全网黄金内容雷达新增的 P0/P1/P2/P3 生产落地功能。

## P0：代理适配器向导

代理向导用于验证方案 A，也就是合法本地代理适配器链路。

它会检查并解释：

1. 本地代理端口；
2. 本地桥端口；
3. 手机是否可以访问这台电脑；
4. 是否已经观察到 `/mp/getappmsgext` 指标接口；
5. 是否已经观察到 `/mp/appmsg_comment` 评论接口。

代理向导本身不会主动抓取隐私流量。它只记录用户可见的就绪信号，并生成带修复建议的报告。

## P1：回放中心和健康监控

回放中心支持脱敏 JSON、JSON 数组和 JSONL 记录。它的作用是：不用每次依赖真实手机会话，也能测试接入、评分、报告和界面。

健康监控汇总：

- 手机就绪状态；
- 代理就绪状态；
- 本地桥就绪状态；
- 数据库就绪状态；
- 任务队列状态。

这样生产运行时不会变成黑盒，用户能看到系统到底卡在哪一层。

## P2：评分工作台和画像分析

评分工作台允许用户调整：

- 阅读权重；
- 点赞权重；
- 评论权重；
- 在看权重；
- 原创权重。

它会预览评分公式和当前记录排名，并生成简洁的账号画像、关键词/分类画像，帮助用户判断哪些账号或方向值得继续跟踪。

## P3：工作区、报告和隐私中心

工作区页面允许用户填写业务工作区名称，并生成 Markdown 决策报告。

诊断快照会导出结构化状态，但不包含私密凭证或原始抓包。

隐私中心明确生产边界：

- 不保存 Cookie；
- 不保存 Header；
- 不保存 Token；
- 不保存证书；
- 不保存原始抓包；
- ADB 只打开用户队列里的文章链接；
- 本地桥只接收脱敏 JSON。

## 推荐生产流程

1. 先运行手机诊断；
2. 再运行代理向导；
3. 确认指标接口和评论接口已经命中；
4. 保存一份脱敏回放样本；
5. 用回放中心复现接入链路；
6. 在评分工作台调整爆款分；
7. 生成工作区报告；
8. 团队交接或生产部署前查看隐私中心。
