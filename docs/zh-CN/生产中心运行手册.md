# 生产中心运行手册

本文说明 Premium Content Radar 面向生产落地的 P0/P1/P2/P3 功能集。

## P0：接入向导和数据质量

### 代理适配器向导

代理向导用于验证方案一：合法本地代理适配器链路。它会检查：

1. 本地代理端口，
2. 本地桥端口，
3. 手机是否能访问这台电脑，
4. 是否观察到 `/mp/getappmsgext` 指标接口，
5. 是否观察到 `/mp/appmsg_comment` 评论接口。

向导本身不会抓取隐私流量。它只记录用户可见的就绪信号，并生成面向修复的报告。

### 数据质量

评分或导出报告前，先用数据质量页检查文章库是否可信：

- 重复链接，
- 缺失标题，
- 缺失账号，
- 缺失链接，
- 异常指标比例，例如点赞数或评论数大于阅读数，
- 无效发布时间。

出现阻塞或风险项时，应该先修数据，再做业务判断。

## P1：回放、健康、趋势和队列可观测性

回放中心支持脱敏 JSON、JSON 数组或 JSONL 记录。它用于在不反复依赖真实手机会话的情况下，测试接入、评分、报告和界面。

健康监控汇总：

- 手机就绪状态，
- 代理就绪状态，
- 本地桥就绪状态，
- 数据库就绪状态，
- 任务队列状态。

趋势追踪会比较同一文章链接的多次快照，显示阅读增长，并给出优先拆解、继续观察或降低优先级的建议。这样系统就不只是静态排行表，而能支持早期爆款发现。

## P2：评分工作台和本地内容分析

评分工作台允许运营者调整：

- 阅读权重，
- 点赞权重，
- 评论权重，
- 在看权重，
- 原创权重。

它会预览评分公式和当前记录排序，并生成简洁的账号、关键词洞察。

内容分析先运行确定性的本地规则，再预留 AI 集成空间。它检查：

- 标题长度，
- 价值/冲突/清单类触发词，
- 互动密度。

这个页面为 AI-ready 设计，但不会伪造模型输出。规则信号弱时，它会明确提示需要人工复核。

## P3：工作区、报告、隐私和交付

工作区页面用于命名业务工作区并生成 Markdown 决策报告。

诊断快照会导出结构化状态，不包含隐私凭证或原始抓包。

隐私中心明确生产边界：

- 不保存 Cookie，
- 不保存请求头，
- 不保存访问令牌，
- 不保存证书，
- 不保存原始抓包，
- 手机调试只打开用户队列里的文章链接，
- 本地桥只接收脱敏 JSON。

交付说明页提供 Windows 和 Linux 安装包交接检查清单，适合在交给非开发人员前使用。

## 推荐生产流程

1. 运行手机诊断。
2. 运行代理向导。
3. 确认指标和评论接口命中。
4. 加载或回放脱敏文章样本。
5. 打开数据质量页并修复风险项。
6. 用趋势追踪找高速增长文章。
7. 在评分工作台调权重。
8. 运行内容分析，检查标题和互动信号。
9. 生成工作区报告。
10. 团队交接前复核隐私中心和交付说明。
