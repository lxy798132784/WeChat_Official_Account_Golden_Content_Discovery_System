# 关键词自动采集运行手册

## 产品目标

主路径是“关键词优先”的自动采集。用户只需要完成：

1. 接入一台已授权 USB 调试的安卓测试手机。
2. 输入市场关键词。
3. 设置爆款文章指标阈值。
4. 点击 **开始关键词自动采集**。

之后软件自动搜索关键词结果页，提取微信公众号文章候选链接，按爆款指标控件筛选，把符合条件的候选文章加入自动采集队列，通过 ADB 在手机上逐篇打开文章，并等待本地代理适配器把真实文章指标发送到本地桥。

## 爆款文章指标控件

关键词发现页提供这些控件：

- **最低阅读**：候选文章入队需要达到的最低 `read_num`。
- **最低点赞**：候选文章入队需要达到的最低 `like_num`。
- **最低评论**：候选文章入队需要达到的最低 `comment_num`。
- **最低爆款分**：候选文章入队需要达到的最低估算爆款分。
- **每词最多候选**：每个关键词搜索页最多解析多少篇候选文章。

注意：搜索页不一定暴露真实阅读、点赞、评论。如果搜索阶段拿不到这些指标，可以先把阈值设为 0，让候选文章自动打开；真正的爆款判断在代理捕获 `/mp/getappmsgext` 和 `/mp/appmsg_comment` 后完成。

当前估算爆款分公式：

```text
hot_score = read_num + like_num * 20 + comment_num * 50
```

## 推荐生产配置

首次验证建议：

```text
最低阅读：0
最低点赞：0
最低评论：0
最低爆款分：0
每词最多候选：5
```

这用于先验证搜索、入队、ADB 打开、代理捕获和数据库入库链路。

确认代理指标能正常入库后，可以提高阈值：

```text
最低阅读：10000
最低点赞：100
最低评论：10
最低爆款分：15000
每词最多候选：10
```

不同领域要分开调。垂直 B2B 主题阅读阈值可以低一点；大众娱乐、本地生活、消费品主题可以高一点。

## 操作流程

1. 打开软件。
2. 如果左侧控制中心被关闭，用 **操作 -> 打开控制中心** 或 `Ctrl+B` 重新打开。
3. 进入 **微信接入**。
4. 保存数据库路径、插件目录、本地桥端口、代理和 ADB 配置。
5. 启动本地代理适配器，确认它能向 `127.0.0.1:9000` 发送精简 JSON。
6. 接入安卓测试手机，并允许 USB 调试。
7. 进入 **关键词发现**。
8. 输入关键词，每行一个。
9. 设置爆款文章指标控件。
10. 点击 **开始关键词自动采集**。
11. 观察 **运行日志**、**自动采集** 和文章表格。

## 现在已经自动化的部分

点击按钮之后，软件会自动：

- 检查 `adb devices`。
- 搜索关键词结果页。
- 解析微信公众号文章 URL。
- URL 去重。
- 应用爆款文章指标控件。
- 把符合条件的 URL 加入自动采集队列。
- 启动调度器。
- 通过 ADB 在手机上打开文章 URL。

真实阅读、点赞、评论仍由本地代理适配器提供，因为这些字段最可靠的来源是文章在已授权手机/浏览器环境中加载时产生的微信接口。

## 故障排查

### 找不到手机

运行：

```bash
adb devices
```

期望看到：

```text
SERIAL_NUMBER    device
```

如果是 `unauthorized`，解锁手机并允许 USB 调试。

### 搜索候选为 0

可能原因：

- 搜索接口拦截自动访问或返回验证码页。
- 网络或代理失败。
- 关键词太窄。
- 搜索页结构变化。

处理方式：

- 换更宽泛的关键词。
- 先用较小的“每词最多候选”验证。
- 用 **生成搜索入口** 调试。
- 必要时用 **导入结果 JSON** 作为备用适配器路径。

### 手机能打开文章但软件没入库

说明 ADB 打开成功，但代理桥没有拿到指标。检查：

- 手机 Wi-Fi 代理是否指向本地代理主机和端口。
- 如果使用 HTTPS 解密，代理证书是否已被手机信任。
- 适配器是否识别 `/mp/getappmsgext` 和 `/mp/appmsg_comment`。
- 软件本地桥端口是否和适配器目标端口一致。

## 安全说明

- 不要把微信账号密码、cookie、私钥、代理证书密钥保存进仓库。
- 只分析你有权访问和使用的数据。
- 先用小规模限额验证，避免噪声流量和账号风险。
