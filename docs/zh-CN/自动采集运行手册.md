# 自动采集配置与运行手册

## 当前自动采集能力说明

v1.1.0 的自动采集不是绕过微信的无人爬虫，而是一个可控的本地自动化链路：

1. 你准备一批自己有权访问的微信公众号文章 URL。
2. 软件把这些 URL 放入自动采集队列。
3. 软件通过 ADB 在已连接的测试手机上逐篇打开文章。
4. 手机端微信加载文章时，本地代理适配器捕获文章指标接口。
5. 代理适配器把脱敏后的精简 JSON 发到软件本地桥。
6. 软件自动入库、评分、展示和导出。

也就是说，软件现在已经可以做到：配置好测试手机、代理和 URL 队列后，用户不需要逐篇手动打开文章。

但软件不会保存微信账号、密码、Cookie、请求头、证书或原始抓包，也不会直接登录微信。

---

## 一、需要准备什么

### 1. 一台测试手机

建议使用专门测试手机，不建议使用主力手机。

手机需要：

- 已安装微信。
- 已登录你自己的微信环境。
- 可以正常打开公众号文章。
- 允许 USB 调试。
- 手机和电脑可以处于同一网络，方便配置本地代理。

### 2. ADB 工具

ADB 用于让电脑命令手机打开文章 URL。

#### Windows 安装

1. 下载 Android Platform Tools。
2. 解压到例如：

```text
C:\platform-tools
```

3. 把这个目录加入系统 PATH。
4. 打开 PowerShell，执行：

```powershell
adb version
```

能看到版本号就说明安装成功。

#### Linux 安装

Ubuntu / Debian：

```bash
sudo apt update
sudo apt install -y adb
adb version
```

#### macOS 安装

如果有 Homebrew：

```bash
brew install android-platform-tools
adb version
```

### 3. 手机开启 USB 调试

不同手机略有差异，通常是：

1. 手机进入“设置”。
2. 进入“关于手机”。
3. 连续点击“版本号”或“构建号”7 次，开启开发者选项。
4. 返回设置，进入“开发者选项”。
5. 打开“USB 调试”。
6. 用 USB 线连接电脑。
7. 手机弹出“是否允许 USB 调试”，选择允许。

电脑执行：

```bash
adb devices
```

正常结果类似：

```text
List of devices attached
ABCDEF123456    device
```

如果显示 `unauthorized`，说明手机上还没有确认授权；拔插 USB 后重新允许。

### 4. 本地代理适配器

自动打开文章只是第一步，真正的阅读数、点赞数、评论数仍然要靠本地代理适配器进入软件。

推荐组合：

```text
mitmproxy / Reqable / Charles / Fiddler / whistle
        ↓
捕获 /mp/getappmsgext 和 /mp/appmsg_comment
        ↓
转换为精简 JSON
        ↓
发送到 127.0.0.1:9000
        ↓
全网黄金内容雷达自动入库
```

如果你还没有选代理工具，优先建议：

- 新手：Reqable。
- 自动化脚本：mitmproxy。
- Windows 桌面习惯：Fiddler Everywhere。
- macOS 桌面习惯：Charles 或 Proxyman。

详细代理配置见：

```text
docs/zh-CN/微信数据接入方案ABC.md
```

## 二、软件内怎么配置自动采集

### 第一步：打开软件

启动 `premium-content-radar`。

左侧控制中心现在有一个新页：

```text
自动采集
```

### 第二步：确认微信接入页

进入“微信接入”页，确认：

- SQLite 数据库路径正确。
- 插件目录正确。
- 本地桥端口通常保持 `9000`。
- ADB 自动化可以先保持关闭。
- 点击“保存运行配置”。
- 点击“加载插件”。
- 点击“发送本地桥冒烟载荷”。

如果文章表能增加测试记录，说明本地桥没问题。

### 第三步：准备文章 URL

进入“自动采集”页，在文章 URL 列表里每行粘贴一个微信文章链接。

示例：

```text
https://mp.weixin.qq.com/s/xxxxxxxxxxxxxxxxxxxxxx
https://mp.weixin.qq.com/s/yyyyyyyyyyyyyyyyyyyyyy
https://mp.weixin.qq.com/s/zzzzzzzzzzzzzzzzzzzzzz
```

也支持包含参数的文章链接，例如：

```text
https://mp.weixin.qq.com/s?__biz=xxx&mid=xxx&idx=1&sn=xxx
```

软件只接受微信文章链接，不接受普通网站链接。

### 第四步：添加 URL 到队列

点击：

```text
添加 URL
```

软件会把合法链接加入队列表。重复链接会跳过。

队列表字段：

| 字段 | 含义 |
|---|---|
| 状态 | pending、opening、opened、failed |
| 尝试 | 当前 URL 已尝试打开次数 |
| 账号 | 预留字段，后续可从导入文件补充 |
| 分类 | 预留字段，后续可从导入文件补充 |
| 上次尝试 | 最近一次 ADB 打开时间 |
| 链接 / 错误 | URL 或失败原因 |

### 第五步：打开自动化安全开关

勾选：

```text
允许本次会话使用 ADB 打开 URL
```

这是显式安全开关。没有勾选时，软件不会执行 ADB 自动打开。

### 第六步：先测试一条

点击：

```text
立即执行下一条
```

软件会执行类似命令：

```bash
adb shell am start -a android.intent.action.VIEW -d "https://mp.weixin.qq.com/s/xxxx"
```

预期结果：

- 手机自动打开微信文章。
- 本地代理看到微信文章接口请求。
- 代理适配器把指标 JSON 发给软件。
- 软件文章表新增或更新记录。

如果手机没有反应，先执行：

```bash
adb devices
```

确认设备在线。

### 第七步：开始自动采集

确认单条测试成功后，再点击：

```text
开始自动采集
```

软件会按“打开间隔”逐篇打开队列中的 URL。

建议初始参数：

```text
打开间隔：30 秒
每条 URL 最大尝试次数：3
```

不要一开始设置太快。微信文章加载、代理捕获和软件入库都需要时间。

### 第八步：停止自动采集

点击：

```text
停止
```

软件会停止定时打开下一篇文章。已经打开的文章不会被撤回。

## 三、自动采集完整链路检查

### 1. 检查 ADB

```bash
adb devices
```

必须看到：

```text
device
```

### 2. 检查手机代理

用手机浏览器打开任意网页，代理工具里应该能看到请求。

### 3. 检查微信文章能打开

在手机微信里手动打开一篇公众号文章，确认能加载。

### 4. 检查软件本地桥

软件里点击：

```text
发送本地桥冒烟载荷
```

文章表应该出现测试记录。

### 5. 检查自动打开

自动采集页点击：

```text
立即执行下一条
```

手机应该自动打开文章。

### 6. 检查真实指标入库

代理适配器捕获到 `/mp/getappmsgext` 后，软件文章表应该出现真实文章记录。

## 四、常见问题排查

### 问题 1：点击“立即执行下一条”没有反应

原因通常是 ADB 没连上。

检查：

```bash
adb devices
```

如果没有设备：

- 检查 USB 线。
- 检查手机是否开启 USB 调试。
- 检查手机是否弹出授权确认。
- Windows 检查手机驱动。

### 问题 2：手机打开了文章，但软件没有新数据

这说明 ADB 开文成功，但代理入库链路没通。

检查顺序：

1. 代理工具是否正在运行。
2. 手机 Wi-Fi 是否配置了电脑 IP 和代理端口。
3. 代理证书是否安装并信任。
4. 代理工具里是否看到 `/mp/getappmsgext`。
5. 适配器脚本是否把 JSON 发送到 `127.0.0.1:9000`。
6. 软件是否已加载 WeChat Provider 插件。

### 问题 3：队列显示 failed

点击失败通常来自 ADB 命令失败。

可能原因：

- adb 不在 PATH。
- 设备未连接。
- 设备 unauthorized。
- 手机不允许从外部 intent 打开链接。
- URL 不是合法微信文章链接。

### 问题 4：代理里有数据，但字段不完整

优先检查适配器 JSON 是否包含这些字段：

```json
{
  "endpoint": "/mp/getappmsgext",
  "title": "文章标题",
  "url": "文章链接",
  "account_name": "公众号名称",
  "gzh_id": "公众号原始ID",
  "category": "分类",
  "publish_time": "2026-06-05T08:30:00Z",
  "read_num": 10000,
  "like_num": 300,
  "old_like_num": 80,
  "comment_num": 25,
  "article_count_30d": 10
}
```

其中 `url` 必须有，否则软件无法去重入库。

### 问题 5：能打开第一篇，后面不动

检查：

- 自动采集是否仍处于启动状态。
- 队列里是否还有 pending 或 failed 且尝试次数未超过上限的任务。
- 打开间隔是否设置过长。
- 日志中是否出现 ADB 错误。

## 五、安全边界

自动采集只做“打开你提供的文章 URL”。它不应该：

- 保存微信账号密码。
- 保存 Cookie。
- 保存请求 Header。
- 保存代理证书。
- 保存原始抓包文件。
- 自动绕过访问控制。
- 自动扫描你没有权限访问的数据。

推荐只保存：

- 文章 URL。
- 标题。
- 公众号名称。
- 发布时间。
- 阅读数。
- 点赞数。
- 在看数。
- 评论数。
- 分类。
- 入库时间。

## 六、推荐生产使用流程

第一次配置：

```text
1. 安装 ADB
2. 连接测试手机
3. 配置手机代理
4. 启动代理适配器
5. 启动全网黄金内容雷达
6. 保存微信接入配置
7. 加载插件
8. 发送本地桥冒烟载荷
9. 自动采集页添加 URL
10. 勾选 ADB 安全开关
11. 立即执行下一条
12. 确认文章入库
13. 开始自动采集
```

日常运行：

```text
1. 插上测试手机
2. 启动代理适配器
3. 启动软件
4. 加载或粘贴 URL 队列
5. 点击开始自动采集
6. 采集完成后导出 CSV/JSON
```

## 七、当前版本限制

v1.1.0 自动采集预览版已经能自动打开 URL 队列，但还有限制：

1. 目前自动发现公众号最新文章列表还没有实现。
2. 当前队列主要靠用户提供 URL 或导入队列文件。
3. 指标捕获仍依赖本地代理适配器。
4. ADB 自动化只打开文章 URL，不做复杂 UI 识别。
5. 不建议高频率批量打开，建议 30 秒以上间隔。

后续可以继续升级：

- 公众号主页文章发现器。
- 自动从种子池生成待抓 URL。
- 更强的代理适配器脚本。
- 自动标记“已入库 / 未捕获 / 需重试”。
- 每日上限、时间窗口和更细限速策略。
