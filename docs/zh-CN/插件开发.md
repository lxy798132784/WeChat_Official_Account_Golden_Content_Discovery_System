# 插件开发

## 插件接口

插件需要实现 `IContentProvider`：

```cpp
class MyProvider final : public QObject, public IContentProvider {
  Q_OBJECT
  Q_PLUGIN_METADATA(IID IContentProvider_iid)
  Q_INTERFACES(IContentProvider)
 public:
  QString providerId() const override;
  QString displayName() const override;
  bool start(QString* errorMessage) override;
  void stop() override;
  QVector<ContentRecord> drainRecords() override;
};
```

## 生命周期

1. 主程序扫描插件目录。
2. `QPluginLoader` 创建插件实例。
3. 主程序把实例转换为 `IContentProvider`。
4. 主程序调用 `start()`。
5. 主程序每秒调用 `drainRecords()`。
6. 程序关闭或插件生命周期结束时调用 `stop()`。

## 记录字段

插件尽量填充以下字段：

- `title`：文章标题。
- `url`：文章链接，也是 SQLite 去重键。
- `accountName`：公众号名称。
- `gzhId`：公众号 ID。
- `category`：分类。
- `readNum`：阅读数。
- `likeNum`：点赞数。
- `oldLikeNum`：看一看或旧点赞数。
- `commentNum`：评论数。
- `articleCount30d`：近三十天发文数。
- `publishedAt`：发布时间。

## 构建集成

插件需要链接 `radar_core`，最终共享库放到运行时插件目录。项目自带的 `WeChatProviderPlugin` 是参考实现。

## 安全要求

- 凭证必须放在用户本地环境或用户配置里，不能写进插件源码。
- 错误通过 `errorMessage` 和日志暴露。
- 不要阻塞 UI 线程。
- 默认只使用 localhost 或用户选择的本地路径。
