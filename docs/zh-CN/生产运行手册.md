# 生产运行手册

## 生产定义

这里的生产落地，是指在用户本地机器上运行桌面主程序，由合法、本地、用户可控的数据适配器把微信公众号文章指标发送到 localhost 桥。软件负责本地存储、评分、筛选和导出。

## 上线前检查

1. 确认操作者拥有数据来源权限。
2. 确认桥端口只监听 `127.0.0.1`，没有暴露到局域网或公网。
3. 确认 SQLite 数据库路径在可靠磁盘上。
4. 确认导出目录符合团队数据管理要求。
5. 在手机方案没有验证前，保持 ADB 自动化关闭。
6. 在目标机器运行 `./scripts/verify-all.sh`。
7. 接入真实适配器前，先运行 `--bridge-smoke`。

## 运行步骤

1. 启动软件。
2. 在 WeChat 页面保存运行配置。
3. 加载插件。
4. 发送本地桥冒烟载荷。
5. 启动合法本地适配器。
6. 观察日志页的入库数量。
7. 检查文章表格、评分和筛选结果。
8. 导出 CSV 或 JSON 给后续分析流程。

## 监控点

需要关注：

- 日志页是否有供应商启动失败。
- 文章数量是否持续增长。
- SQLite 数据库文件大小和磁盘剩余空间。
- 下游使用导出文件时，是否需要校验 SHA256。

## 备份

建议备份：

- SQLite 数据库。
- 运行配置 JSON。
- 导出的 CSV 和 JSON 文件。

不要把账号凭证、Cookie 或私有流量备份到 Git 仓库。

## 异常处理

如果发现数据不对：

1. 先停止本地适配器。
2. 导出当前记录作为证据。
3. 检查适配器发送的 JSON 样本。
4. 运行 `--bridge-smoke` 验证主程序桥链路。
5. 运行单元测试验证解析器。
6. 修正载荷协议后再恢复采集。

## 需要大王配合的内容

- 提供真实运行目标系统：Linux、Windows，还是两者都要。
- 提供合法可用的数据来源方式，或者允许我们写本地适配器对接现有代理。
- 确认生产数据库目录和备份策略。
- 确认是否允许 ADB 自动化控制测试手机。
- 如果要 Windows 正式包，需要一台装好 Qt 6 的 Windows 构建环境或 GitHub Windows 构建配置。
