# 手机接入诊断运行手册

## 目标

手机接入诊断的目标，是让普通用户也能看懂关键词自动采集为什么能启动、为什么不能启动。软件不再只提示“ADB 失败”，而是逐层检查并告诉用户应该修哪一层。

生产主路径是：

1. 接入一台已授权 USB 调试的安卓测试手机。
2. 输入关键词。
3. 配置爆款文章指标。
4. 点击开始关键词自动采集。
5. 软件自动搜索候选、按指标筛选、在手机上打开文章，并通过本地代理桥接收真实指标。

手机诊断就是这条链路的启动前保护。

## 诊断层级

### 1. ADB 工具

检测命令：

```bash
adb version
```

作用：

确认电脑已经安装 ADB，并且软件可以执行 ADB。如果这里失败，软件无法检测或控制任何安卓手机。

修复方法：

- 安装 Android Platform Tools。
- 把 adb 所在目录加入 PATH。
- Windows 用户还需要安装手机厂商 USB 驱动或 Google USB Driver。

### 2. ADB 服务

检测命令：

```bash
adb start-server
adb devices -l
```

作用：

ADB 工具存在不代表 ADB 服务正常。服务可能没启动、卡住、端口异常或状态过期。

界面控件：

```text
重启 ADB
```

它会执行：

```bash
adb kill-server
adb start-server
```

### 3. 手机连接

检测命令：

```bash
adb devices -l
```

软件会识别这些状态：

- `device`：手机已连接并已授权。
- `unauthorized`：手机已连接，但没有授权 USB 调试。
- `offline`：ADB 能看到手机，但暂时不能控制。
- 无设备：ADB 看不到手机。
- 多设备：需要在界面里选择目标手机。

### 4. USB 授权

如果状态是 `unauthorized`，处理方法：

1. 解锁手机。
2. 打开开发者选项。
3. 打开 USB 调试。
4. 拔插 USB。
5. 在手机弹窗中点击允许。
6. 如果有“始终允许此电脑”，建议勾选。

### 5. Shell 控制

检测命令：

```bash
adb -s SERIAL shell getprop ro.product.model
adb -s SERIAL shell getprop ro.build.version.release
adb -s SERIAL shell echo ok
```

作用：

关键词自动采集不只是要看到手机，还要能控制手机打开文章链接。所以必须确认 shell 命令可执行。

### 6. 打开文章链接

界面控件：

```text
测试打开链接
```

执行命令：

```bash
adb -s SERIAL shell am start -a android.intent.action.VIEW -d "https://mp.weixin.qq.com/s/test"
```

注意：

这个测试会改变手机屏幕，所以全量诊断里提供了“是否包含打开链接测试”的勾选项。默认可以先跳过，需要验证完整链路时再打开。

### 7. 本地代理端口

如果用户填写代理端口，软件会检测电脑本机：

```text
127.0.0.1:端口
```

是否可以连接。

这只能说明代理进程在电脑上监听，不等于手机流量已经走代理。手机 Wi-Fi 代理仍然需要用户按本地代理工具说明配置。

### 8. 本地桥端口

软件会检测配置的本地桥端口是否可达。本地桥负责接收代理适配器发来的精简脱敏 JSON。

在“微信接入”页面可以点击本地桥冒烟按钮，发送模拟 `/mp/getappmsgext` 和 `/mp/appmsg_comment` 载荷，验证解析和入库链路。

## 不同系统的处理方法

### Windows

需要安装：

- Android Platform Tools
- 手机厂商 USB 驱动或 Google USB Driver

然后打开设备管理器，确认手机显示为 Android ADB Interface，而不是 Unknown Device。

### Linux

安装 `android-tools-adb`。如果出现权限问题，需要配置 udev rules。

常用命令：

```bash
sudo usermod -aG plugdev $USER
sudo udevadm control --reload-rules
sudo udevadm trigger
adb kill-server
adb start-server
```

### macOS

安装 Android Platform Tools。通常不需要厂商 USB 驱动。重点检查 USB 线是否支持数据传输，以及手机是否信任这台电脑。

## 启动前门禁

用户点击“开始关键词自动采集”时，软件现在会先运行手机预检。只有下面核心项通过，才允许启动采集：

- ADB 工具
- ADB 服务
- 手机连接
- USB 授权
- Shell 控制
- 目标设备选择

如果预检失败，软件会自动切到手机诊断页，并显示阻塞原因。

## 报告功能

界面提供：

- 复制报告：复制文本诊断报告，方便排障。
- 导出 JSON：导出结构化诊断报告，包含状态、目标设备、设备列表、详情、修复建议和原始输出。

不要把 Cookie、Token、证书、原始微信抓包内容放进诊断报告。
